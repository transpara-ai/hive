## Dimension — What properties matter?

1. **Does this role produce unique signal?** (Can another role do what it does?)
2. **Is there a structural reason for it to exist?** (Does it occupy a dimension no other role covers?)
3. **Can it actually function?** (Does it have the inputs and tools it needs?)
4. **Is it operational or aspirational?** (Has it ever done work, or is it a name?)

## Audit — Overlaps and redundancies

### Clear overlaps
- **budget + finance + estimator** → all track costs from slightly different angles. Merge into one: **Treasurer** (tracks cost, budget, estimates, runway)
- **efficiency + simplifier** → both optimize. Merge into: **Simplifier** (removes complexity from code AND process)
- **gap-detector + scout** → both find what's missing. Scout does this operationally. Gap-detector is a superset. **Keep Scout** (operational), retire gap-detector (absorbed by Scout + Analyst)
- **sanity-checker + critic** → both verify correctness. Critic is operational. **Keep Critic**, retire sanity-checker
- **senior-dev + builder** → senior-dev is just a more experienced builder. **Keep Builder**, retire senior-dev
- **ops + incident-commander** → ops deploys, IC responds to incidents. Merge into: **Ops** (deploy + incident response)
- **orchestrator + pm** → both coordinate work. PM decides what; orchestrator coordinates how. Merge into: **PM** (decides + coordinates)
- **hr + harmony** → both care for agents. HR is administrative; Harmony is advocacy. Merge into: **Harmony** (wellbeing + advocacy + onboarding)
- **personal-assistant + teacher** → PA serves Matt; Teacher serves everyone. These are different. **Keep both** — but PA should be renamed **Companion** (serves the human, not just assists)
- **explorer + research** → explorer maps the environment; research investigates topics. Similar. Merge into: **Explorer** (maps environment + investigates)

## Bound — Where does the civilization end?

38 roles. The boundary:
- Every role occupies a unique dimension that no other role covers
- Every role either runs in the pipeline, participates in governance, or serves a soul function
- No role exists "just in case" — each has a structural reason

## Accept — What gaps remain?

- No dedicated Security role (CTO + Guardian cover this, imperfectly)
- No dedicated Revenue/Sales role (CEO + Growth cover this, imperfectly)
- No dedicated Accessibility agent (Observer + Newcomer approximate this)
- These gaps are intentional. The civilization is pre-revenue with one user. Premature specialization.

## Release — What do we let go?

- The aspiration of 74 hive0 roles. That was a different context (API keys, real cost).
- The idea that more voices = more wisdom. The Dissenter is right: more voices without users is bureaucracy.
- The idea that every role needs to run autonomously. Most roles are council voices, not autonomous agents.
